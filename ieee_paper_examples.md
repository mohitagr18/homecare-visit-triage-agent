# Case Studies for IEEE Paper

The following concrete examples demonstrate how the four-layer testing architecture actively prevents leakage, detects hallucinations, and halts on ambiguity. These examples can be directly adapted into the "Results" or "Case Studies" subsections of the IEEE paper.

---

## Case Study 1: Preventing PHI Leakage via One-Way Bridge (Layer 1 & 2)

**The Scenario:** 
A human annotator creates a ground truth record with the physical file name `H.Leal-122425-123025.pdf` (containing real PHI). The VLM extracts data from the anonymized file `patient_c_week3.pdf`. To score the extraction, the pipeline must mathematically compare them without writing `H.Leal` to the final metrics report.

**The Test Case (`test_output_contains_only_anonymized_filenames`):**
The Integration test suite intercepts the final `failures.json` and `paper_table.md` artifacts generated by the LangGraph workflow and runs a strict substring assertion against a predefined list of real patient names.

**The Execution Example:**
1. **Input:** Extracted `patient_c` data enters the pipeline.
2. **Internal Resolution:** `NameResolver` dynamically maps `patient_c_week3.pdf` -> `H.Leal` purely in-memory to fetch the ground truth data.
3. **Evaluation Output:** The pipeline writes the failure report:
```json
{
  "source_file": "patient_c_week3.pdf",
  "row_index": 2,
  "failed_fields": [
    {"field": "total_hours", "predicted": 8.0, "expected": 7.5}
  ]
}
```
4. **Test Verification:** The integration test asserts `assert "Leal" not in output_json`. The test **passes**, definitively proving that the internal lookup bridge acts as a one-way street—PHI is used for logic but structurally prohibited from serializing to disk.

## Case Study 2: Detecting Temporal Hallucinations (Layer 4)

**The Scenario:** 
LLMs frequently suffer from "temporal drift" or hallucinations when reading handwriting (e.g., misreading a year as 2025 instead of 2026). Traditional pipelines would simply mark this as a "Missed Row" resulting in 0% accuracy without explaining why.

**The Execution Example:**
During the batch evaluation of the `vlm_full_page` methodology, the pipeline evaluated `patient_p_week16.pdf`. The model hallucinated the year, extracting dates like `2026-12-25`.

**The Pipeline Response:**
Because the pipeline maps the anonymized ID to the true Ground Truth file, it knows the temporal bounds of that specific patient's care. It detected that the extracted dates completely failed to intersect with *any* known ground truth dates for that patient, emitting a structural failure warning rather than silently degrading the score:

> `WARNING: DATA ANOMALY: patient_p_week16.pdf resolved to a known patient, but 0/10 extracted dates matched GT. Extracted dates: ['2026-12-25', '2026-12-26']. This likely indicates an extraction pipeline error (e.g., misread year/month).`

**Significance:** The pipeline successfully categorized a failure not just as "wrong hours", but specifically as an upstream temporal hallucination.

## Case Study 3: Halting on Ambiguity via HITL (Layer 3)

**The Scenario:** 
An OCR pipeline flags a signature on a timesheet as "unclear" but the extracted math (8.0 hours) perfectly matches the Ground Truth (8.0 hours). Should the pipeline report this as a success or a failure? 

**The Test Case (`test_ambiguous_rows_trigger_interrupt`):**
If a system encounters conflicting signals (Validator says "Flagged", but Evaluator says "GT Matched"), autonomous AI agents often guess. The Layer 3 test explicitly prohibits guessing.

**The Execution Example:**
1. The synthetic test injects a row into the LangGraph state:
```json
{
    "total_hours": 8.0, 
    "gt_hours": 8.0, 
    "status": "flagged",
    "issues": "Signature unclear"
}
```
2. The `triage_decision` edge function detects the conflict.
3. **Test Verification:** The test asserts `assert snapshot.next == ("human_review",)`. The test **passes**, proving the LangGraph execution physically halts. 
4. The system waits indefinitely until a human payload is injected (`Command(resume=[{"accept": True}])`), proving the trust boundary is mechanically enforced by the graph state, rather than just being a policy guideline.
